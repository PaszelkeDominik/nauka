Życie w pudełku: Box model (padding / border / margin)

Box model widzieliśmy już w narzędziach developerskich. Składa się z: margin | border | padding | element.
Padding jest to wewnętrzny margines, odpowiada za oddalenie o elementu. Możemy określić odległość dla każdego kierunku (top right bottom left). Jest możliwy zapis w jednym wierszu poprzez padding: 10px 15px 20px 10px;
Border – obramowanie. Podajemy border-width: 10px; border-color: red; border-style: solid; Istnieje możliwość zapisu w jednym wierszu: border: 10px red solid; (kolejność obojętna, przeglądarka sama zinterpretuje odpowiednio) border-radius: 50%; - obramowanie w kształcie koła. Radius zaokrągla o podana wartość;
margines to margines:) zapis analogiczny jak w padding-u;
Własność margin: 10px auto; centruje nam na środku – uwaga musimy podać width żeby margin auto miało punkt odniesienia.

Życie w pudełku: box-sizing

Box-sizing: border-box; Ta własność definiuje że wartość podana w width określa odległość dla border+padding+content. Domyślnie ustawienie to content-box; i definiuje odległość tylko dla contentu.

Życie w pudełku: position

Position: static; -wartość domyślna;
position: relative; oraz position: absolute; samo nic nie zmienia, ale daje nowe możliwości. Za pomocą własności top,right,bottom, left określamy odległości wzgledem relativ.
z-index definiuje kolejność wyświetlania nachodzących elementów. Z-index z większą wartością jest na wierzchu.
position:fixed ustawia element w określonym miejscu na zawsze. Przykleja do ekranu.
positon:sticky; określa położenie w określonym elemencie i przykleja się do ekranu tak długo jak jest widoczny ten element.

Życie w pudełku: float

Float nie służy do budowania layoutów!
float służy do opływania. Najczęściej używa się przy image. Float: left; ustawia element z lewej strony a treść opływa go z prawej.
clear: both; dopasowuje kontener do float-a
overflow: hidden; tak jak wyżej;

JS: Typy danych i immutability

Trzy sposoby zapisywania zmiennych: let | const | var. Var odchodzi się od jego używania. const – tzn. constant, ma stałe połączenie z jedną wartością. let pozwala nam zmieniać wartość zmiennej.
Typy proste:
String – tekst „” lub ``
Number – liczba
Bigint – duża liczba, większa niż 2 do 53
boolean – true lub false
null – wartość celowo niezdefiniowana
undefined – brak definicji
*symbol – tworzy „unikalne opakowanie”
wyrażenie typeof „nazwaZmiennej” zwraca nam typ;
immutability – niezmienność – nie możemy zmienić wartości primitive values w JS. To co zmieniamy to połączenia między nimi, a zmiennymi.

Block – Element – Modifier

BEM = BLOCK__ELEMENT—MODIFIER
Sposób nazywania klas. Dzielimy kod na Blok potem ten blok na elementy i jeśli jest to koniecznie na modyfikator. Przed elementem stawiamy „__” a przed modyfikatorem ”--”. Stosujemy dwie klasy do jednego elementu jeżeli druga klasa dodaje jakieś specyficzne własności do elementu.
Block – element nadrzendny
Element – część bloku( nie musi być bezpośrednim dzieckiem)
Modifier- jeśli element występuje w kilku wariacjach tworzymy dodatkową klasę.

DOMowe przedszkole

DOM = Document Object Model
” to co widzimy w devtools”. To „złożony w przeglądarce HTML. JavaScript może modyfikować DOM.
Za pomocą funkcji document.querySelector(`.selector`) zwracamy node (węzeł) z HTML-a, selektor jest zapisywany zgodnie z CSS.
const heading = document.querySelector(`.main-heading—js`);
heading.innerHTML = `tekst który podmieniamy w miejscu na które wskazuje .main-heading—js` ;
innerHTML – wewnętrzny HTML
Pro tip – korzystając z BEM – tworzymy dodatkową klasę z modyfikatorem --js (ułatwia to identyfikację / nie dopuszcza to potencjalnych problemów)
