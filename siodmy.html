Pseudoelementy & Pseudoselektory

Za pomocą, dodania do selektora odpowiedniego słowa klucz, można stylować specyficzną część elementu.

::before & ::after
Nie są elementem DOM. Za ich momoćą możemy stylować, odpowiednio przód i tyłelementu. Aby stworzyć nowy element, wykorzystujemy właściwość content:"...";
::hover
zmienia działanie w momencie najechania kursorem myszy.
::visited
Odnosi się do elementu, który da się odwiedzić, np. link.
::focus
Reprezentuje element typu input. Jest głównie wywoływany poprzez kliknięcie lub klawisz "tab".
::first-child
nadaje własności pod warunkiem, że jest pierwszym dzieckiem rodzica.
::last-child
nadaje własności pod warunkiem, że jest ostatnim dzieckiem rodzica.
::not(.selector)
Sprawia, iż własności są stosowane do elementów, które nie spełniają podanego kryterium. np. (:hover) {własności }

.gitignore

plik .gitignore zawiera zestaw reguł, które pozwalają nam na zdefiniowanie listy rzeczy, które nie powinny znajdować się w naszym repozytorium, np. pliki z hasłami.

Node.js

Node.JS jest platformą umożliwiającą uruchomienie kodu JavaScript. Platforma udostępnia API, którego najszerszym (lecz nie jedynym) zastosowaniem jest tworzenie stron internetowych opartych na programowaniu sterowanym zdarzeniami. Poza samą możliwością uruchomienia skryptów JavaScript w środowisku innym niż przeglądarka, platforma oferuje wiele modułów stanowiących potężny zestaw narzędzi programistycznych. -"to taki javascript uruchamiany poza przeglądarką"

NPM i NPX

npm - to repozytorium z gotowymi paczkami. Pozwala na dodanie do projektów sporo gotowych bibliotek/ narzędzi. Dzięki niemu nie trzeba polegać na konfiguracji edytora/ jego wtyczek. Jest uniwersalnym żródłem ze wszystkimi narzędziami, oraz zapewnia dostęp do aktualizacji.

npm to taka skrzynka z narzędziami, których możemy używać w naszym projekcie.

npm init
npm install nazwa-paczki --save
npm install nazwa-paczki --save-dev
npm install node-sass --save-dev

  • --save opcja do instalacji rzeczy uzywanych w przeglądarce.
  • --save-dev do narzędzi, typowo do rozwoju oprogramowania, np.server lokalny.
package-lock.json
trzyma informacje o tym co zainstalowaliśmy -"blokuje versje".
node_modules
katalog z bibliotekami - nie edytujemy, tych plików! nie komitujemy!!
npx
do korzystania z narzędzi bez ich instalacji.
